libtoxcore-racket

Lehi Toskin

This library provides a Racket-like interface to the libtoxcore, libtoxav, and libtoxdns libraries. Be aware that Tox is currently thread-unsafe, so please take all proper precautions while using the library.

procedure

(bootstrap-from-address tox    
  address    
  ipv6?    
  port    
  public-key) → boolean?
  tox : _Tox-pointer
  address : string?
  ipv6? : boolean?
  port : integer?
  public-key : string?
Resolves address into an IP address. If successful, sends a "get nodes" request to the given node with ip, port (in network byte order, HINT: use htons()) and public_key to setup connections

address can be a hostname or an IP address (IPv4 or IPv6)

if ipv6? is #f, the resolving sticks STRICTLY to IPv4 addresses

if ipv6? is not #f, the resolving looks for IPv6 addresses first, then IPv4 addresses.

returns #t if the address was converted into an IP address

returns #f otherwise

procedure

(callback-connection-status my-tox    
  anonproc    
  userdata) → void?
  my-tox : _Tox-pointer
  anonproc : procedure?
  userdata : cpointer?
This function is kind of tricky because the C library requires a function as a parameter (anonproc). This wrapper procedure is kind of tricky and shouldn’t be considered complete.

anonproc is in the form (anonproc tox friendnumber status userdata)

Status:

0 – friend went offline after being previously online

1 – friend went online

NOTE: This callback is not called when adding friends, thus the "after being previously online" part. it’s assumed that when adding friends, their connection status is offline.

procedure

(send-action tox friendnumber action len) → integer?

  tox : _Tox-pointer
  friendnumber : integer?
  action : string?
  len : integer?
Send an action to an online friend.

return the message id if packet was successfully put into the send queue.

return 0 if it was not.

You will want to retain the return value, it will be passed to your read_receipt callback if one is received.

send-action-withid will send an action message with the id of your choosing, however we can generate an id for you by calling plain send-action.

procedure

(send-message tox friendnumber message len) → integer?

  tox : _Tox-pointer
  friendnumber : integer?
  message : string?
  len : integer?
Send a text chat message to an online friend.

return the message id if packet was successfully put into the send queue.

return 0 if it was not.

maximum length of messages is TOX_MAX_MESSAGE_LENGTH, your client must split larger messages or else sending them will not work.

You will want to retain the return value, it will be passed to your read_receipt callback if one is received.

send-message-withid will send a message with the id of your choosing, however we can generate an id for you by calling plain send-message.

procedure

(tox-new ipv6?) → _Tox-pointer

  ipv6? : boolean?
Run this function at startup.

Initializes a tox structure

The type of communication socket depends on ipv6enabled:

If set to 0 (zero), creates an IPv4 socket which subsequently only allows IPv4 communication

If set to anything else, creates an IPv6 socket which allows both IPv4 AND IPv6 communication

return allocated instance of tox on success.

return 0 if there are problems.

Receive decoded audio frame.

frame-size is the size of dest in frames/samples (one frame/sample is 16 bits or 2 bytes and corresponds to one sample of audio.)
